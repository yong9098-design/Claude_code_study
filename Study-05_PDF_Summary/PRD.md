# PDF 문서 요약기 - PRD (Product Requirements Document)

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 제품명 | PDF 문서 요약기 |
| 버전 | v1.0.0 |
| 작성일 | 2026-03-28 |
| 목표 | PDF 문서를 업로드하면 AI가 자동으로 요약·감정분석·공감메시지를 생성하는 웹 앱 |

---

## 2. 문제 정의

- PDF 문서를 읽는 데 시간이 많이 소요됨
- 핵심 내용 파악이 어려움
- 문서의 감정적 맥락을 빠르게 파악하고 싶음

---

## 3. 사용자 시나리오

1. 사용자가 `index.html`을 브라우저에서 연다
2. PDF 파일을 드래그&드롭 또는 클릭으로 업로드한다
3. 앱이 PDF에서 텍스트를 추출한다
4. OpenRouter AI가 텍스트를 분석한다
5. 결과 화면에 요약, 핵심포인트, 감정분석, 공감메시지가 표시된다

---

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 배포 방식 | 단일 `index.html` 파일, 서버 불필요 |

| 언어 | 한국어 UI |

---

## 6. 기술 스택

| 레이어 | 기술 |
|--------|------|
| PDF 추출 | PDF.js (CDN) |
| AI 모델 | stepfun/step-3.5-flash:free (OpenRouter) |
| API 통신 | Fetch API |
| UI | Vanilla HTML/CSS/JS (단일 파일) |
| 아이콘 | Unicode 이모지 |

---

## 7. 기능 명세서

### API 요청 형식

```json
{
  "model": "stepfun/step-3.5-flash:free",
  "messages": [
    {
      "role": "system",
      "content": "당신은 문서 분석 전문가입니다..."
    },
    {
      "role": "user",
      "content": "[PDF 텍스트 내용]"
    }
  ],
  "temperature": 0.7,
  "max_tokens": 2000
}
```

### API 응답 파싱 형식 (JSON)

```json
{
  "summary": "문서 전체 요약...",
  "keyPoints": ["핵심 포인트 1", "핵심 포인트 2"],
  "sentiment": {
    "tone": "긍정적",
    "emoji": "😊",
    "description": "전반적으로 긍정적인 내용..."
  },
  "empathyMessage": "독자를 위한 공감 메시지..."
}
```

## 8. 오류 처리

| 오류 상황 | 처리 방법 |
|-----------|----------|
| PDF 파일 아님 | 파일 형식 오류 메시지 |
| 텍스트 추출 실패 | 스캔 PDF 안내 메시지 |
| API 키 오류 | API 키 확인 요청 |
| 네트워크 오류 | 재시도 버튼 표시 |
| 응답 파싱 실패 | 원문 텍스트로 폴백 |
