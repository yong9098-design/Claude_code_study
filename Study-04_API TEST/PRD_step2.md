# PRD Step 2 — 레시피 생성

## 개요
Step 1에서 추출한 재료 목록을 기반으로, `google/gemma-3-27b-it:free` 모델을 사용해 실현 가능한 레시피를 생성하고 사용자에게 제공한다.

---

## 목표
- 인식된 재료를 활용해 실제 요리 가능한 레시피를 AI가 자동 생성한다.
- 사용자가 선호 조건(인원수, 요리 시간, 음식 종류)을 설정해 맞춤 레시피를 받을 수 있다.
- 생성된 레시피를 보기 좋게 구조화하여 화면에 표시한다.
- 사용자가 마음에 드는 레시피를 Step 3(프로필)에 저장할 수 있다.

---

## 사용자 스토리
- 사용자는 Step 1에서 확정된 재료 목록을 확인한다.
- 인원수, 요리 시간, 선호 카테고리(한식/양식/중식 등)를 선택한다.
- "레시피 생성" 버튼을 누르면 AI가 레시피를 최대 3개 추천한다.
- 각 레시피 카드를 클릭하면 상세 조리법을 확인할 수 있다.
- "저장" 버튼으로 레시피를 프로필에 저장한다.

---

## 기능 요구사항

### 프론트엔드
| 기능 | 설명 |
|------|------|
| 재료 목록 표시 | Step 1에서 넘어온 재료를 읽기 전용 태그로 표시 |
| 조건 설정 UI | 인원수(1~6), 요리 시간(15/30/60분 이내), 카테고리 선택 |
| 레시피 생성 버튼 | 클릭 시 로딩 표시 후 결과 렌더링 |
| 레시피 카드 목록 | 요리명, 예상 시간, 난이도, 한 줄 소개 표시 (최대 3개) |
| 레시피 상세 모달 | 재료 목록, 단계별 조리법, 팁 표시 |
| 저장 버튼 | 각 레시피 카드에 위치, Step 3 프로필에 저장 |
| 재생성 버튼 | 같은 조건으로 다른 레시피 재요청 |

### 백엔드 API
| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/api/generate-recipes` | POST | 재료 + 조건을 받아 레시피 목록 반환 |

**요청 형식**
```json
{
  "ingredients": ["계란", "우유", "당근"],
  "options": {
    "servings": 2,
    "max_time_minutes": 30,
    "category": "한식"
  }
}
```

**응답 형식**
```json
{
  "recipes": [
    {
      "id": "uuid",
      "name": "당근 계란볶음",
      "time_minutes": 20,
      "difficulty": "쉬움",
      "summary": "간단하게 만드는 고소한 계란볶음",
      "ingredients": [
        {"name": "계란", "amount": "3개"},
        {"name": "당근", "amount": "1/2개"}
      ],
      "steps": [
        "당근을 채 썰어 준비한다.",
        "팬에 기름을 두르고 당근을 볶는다.",
        "계란을 넣어 함께 볶는다."
      ],
      "tip": "불을 약하게 유지하면 계란이 부드럽습니다."
    }
  ]
}
```

### AI 프롬프트 설계
```
다음 재료로 만들 수 있는 요리 레시피를 3가지 추천해주세요.

재료: {ingredients}
인원수: {servings}명
조리 시간: {max_time}분 이내
카테고리: {category}

반드시 아래 JSON 형식으로만 응답하세요:
{
  "recipes": [
    {
      "name": "요리명",
      "time_minutes": 숫자,
      "difficulty": "쉬움|보통|어려움",
      "summary": "한 줄 소개",
      "ingredients": [{"name": "재료명", "amount": "양"}],
      "steps": ["1단계", "2단계"],
      "tip": "요리 팁"
    }
  ]
}
```

## 비기능 요구사항
- API 응답 시간: 20초 이내 (레시피 3개 생성 기준)
- JSON 파싱 실패 시 재시도 로직 1회 적용
- 재료가 0개인 경우 레시피 생성 비활성화 및 안내 메시지 표시

---

## 기술 스택
| 구분 | 기술 |
|------|------|
| 프론트엔드 | Step 1과 동일 (SPA 라우팅 또는 쿼리스트링으로 재료 전달) |
| 백엔드 | Node.js + Express |
| AI 모델 | `google/gemma-3-27b-it:free` via OpenRouter |
| 상태 전달 | Step 1 → Step 2: URL 파라미터 또는 sessionStorage |

---

## 화면 플로우
```
[Step 1에서 넘어온 재료 목록]
        ↓
[조건 설정: 인원수 / 시간 / 카테고리]
        ↓ (레시피 생성 버튼 → 로딩)
[레시피 카드 3개 표시]
        ↓ (카드 클릭)
[상세 조리법 모달]
        ↓ (저장 버튼)
[→ Step 3: 프로필에 저장]
```
