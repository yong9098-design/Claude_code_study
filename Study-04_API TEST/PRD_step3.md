# PRD Step 3 — 사용자 프로필 및 레시피 저장

## 개요
사용자 프로필을 생성하고, Step 2에서 마음에 든 레시피를 저장·관리할 수 있는 기능을 제공한다. 저장된 레시피는 언제든지 조회하고 재활용할 수 있다.

---

## 목표
- 사용자가 회원가입/로그인 없이도 간단한 닉네임으로 프로필을 생성할 수 있다.
- 저장한 레시피를 목록으로 관리(조회/삭제)할 수 있다.
- 저장된 재료 목록을 재사용해 새 레시피를 빠르게 생성할 수 있다.
- 향후 계정 기반 인증으로 확장 가능한 구조로 설계한다.

---

## 사용자 스토리
- 사용자는 닉네임을 입력해 로컬 프로필을 생성한다.
- Step 2에서 "저장" 버튼을 누르면 레시피가 프로필에 저장된다.
- 프로필 페이지에서 저장된 레시피 목록을 확인한다.
- 레시피 카드를 클릭하면 상세 조리법을 다시 볼 수 있다.
- 불필요한 레시피를 삭제할 수 있다.
- 저장된 레시피의 재료 목록을 Step 2로 가져가 새 레시피를 생성한다.

---

## 기능 요구사항

### 프론트엔드
| 기능 | 설명 |
|------|------|
| 프로필 생성 UI | 닉네임 입력 + 아바타 선택 (이모지 또는 색상) |
| 프로필 헤더 | 닉네임, 저장된 레시피 수 표시 |
| 저장 레시피 목록 | 카드 그리드 형태, 요리명/저장일/난이도 표시 |
| 레시피 상세 보기 | Step 2와 동일한 모달 재사용 |
| 레시피 삭제 | 카드의 삭제 아이콘 클릭 → 확인 후 삭제 |
| 재료 재사용 버튼 | 저장된 레시피의 재료를 가지고 Step 2로 이동 |
| 프로필 편집 | 닉네임 수정 |

### 백엔드 API
| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/api/profile` | POST | 프로필 생성 |
| `/api/profile/:id` | GET | 프로필 조회 |
| `/api/profile/:id` | PATCH | 프로필 수정 (닉네임) |
| `/api/profile/:id/recipes` | GET | 저장된 레시피 목록 조회 |
| `/api/profile/:id/recipes` | POST | 레시피 저장 |
| `/api/profile/:id/recipes/:recipeId` | DELETE | 레시피 삭제 |

**프로필 생성 요청**
```json
{
  "nickname": "요리왕홍길동",
  "avatar": "🍳"
}
```

**프로필 응답**
```json
{
  "id": "uuid",
  "nickname": "요리왕홍길동",
  "avatar": "🍳",
  "created_at": "2026-03-22T00:00:00Z"
}
```

**레시피 저장 요청**
```json
{
  "recipe": {
    "name": "당근 계란볶음",
    "time_minutes": 20,
    "difficulty": "쉬움",
    "summary": "간단하게 만드는 고소한 계란볶음",
    "ingredients": [
      {"name": "계란", "amount": "3개"},
      {"name": "당근", "amount": "1/2개"}
    ],
    "steps": ["당근을 채 썰어 준비한다.", "팬에 기름을 두르고 당근을 볶는다."],
    "tip": "불을 약하게 유지하면 계란이 부드럽습니다."
  },
  "source_image_ingredients": ["계란", "우유", "당근"]
}
```

## 데이터 저장 방식

### Phase 1 (MVP) — 파일 기반
- 서버의 `data/profiles.json` 에 프로필과 레시피를 저장
- 별도 DB 없이 빠른 프로토타이핑 가능
- 프로필 ID는 브라우저 `localStorage` 에 보관

### Phase 2 (확장) — DB 전환
- SQLite 또는 PostgreSQL로 마이그레이션
- 이메일/소셜 로그인 추가 가능한 구조 (userId 컬럼 예약)

---

## 비기능 요구사항
- 프로필 ID는 UUID로 생성, localStorage에 저장해 자동 로그인
- 저장 레시피 최대 50개 (MVP 기준)
- 삭제는 소프트 딜리트 없이 즉시 삭제 (MVP)
- 새로고침 후에도 프로필이 유지되어야 한다

---

## 기술 스택
| 구분 | 기술 |
|------|------|
| 프론트엔드 | Step 1·2와 동일 |
| 백엔드 | Node.js + Express |
| 저장소 (MVP) | JSON 파일 (`data/profiles.json`) |
| 저장소 (확장) | SQLite / PostgreSQL |
| 세션 관리 | 브라우저 `localStorage` (profileId 보관) |

## 완료 기준 (Definition of Done)
- [ ] 닉네임 입력으로 프로필이 생성되고 localStorage에 ID가 저장된다.
- [ ] Step 2 "저장" 버튼으로 레시피가 서버에 저장된다.
- [ ] 프로필 페이지에서 저장된 레시피 목록이 조회된다.
- [ ] 레시피 상세 모달이 정상 표시된다.
- [ ] 레시피 삭제가 동작한다.
- [ ] 재료 재사용 버튼으로 Step 2에 재료가 전달된다.
- [ ] 새로고침 후에도 프로필이 유지된다.
