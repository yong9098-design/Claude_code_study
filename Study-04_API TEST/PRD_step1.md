# PRD Step 1 — 냉장고 이미지 인식

## 개요
사용자가 냉장고 사진을 업로드하면, OpenRouter API의 `google/gemma-3-27b-it:free` 모델을 사용해 이미지에서 식재료를 인식하고 목록으로 반환한다.

---

## 목표
- 사용자가 냉장고 사진을 웹에서 간편하게 업로드할 수 있어야 한다.
- AI 모델이 이미지를 분석해 식재료를 정확하게 추출한다.
- 추출된 재료 목록을 구조화된 형태(JSON)로 반환해 다음 단계에서 활용할 수 있어야 한다.

---

## 사용자 스토리
- 사용자는 냉장고 사진을 드래그&드롭 또는 파일 선택으로 업로드한다.
- 업로드 후 "재료 인식" 버튼을 누르면 AI가 분석을 시작한다.
- 분석 완료 후 화면에 인식된 재료 목록이 표시된다.
- 사용자는 목록에서 재료를 추가/삭제하여 보정할 수 있다.

---

## 기능 요구사항

### 프론트엔드
| 기능 | 설명 |
|------|------|
| 이미지 업로드 UI | 드래그&드롭 + 파일 선택 지원, 미리보기 표시 |
| 지원 파일 형식 | JPG, PNG, WEBP (최대 10MB) |
| 재료 인식 버튼 | 업로드 후 활성화, 로딩 스피너 표시 |
| 재료 목록 표시 | 인식된 재료를 태그 형태로 표시 |
| 재료 편집 | 태그 추가(입력창) / 삭제(x 버튼) |
| 다음 단계 버튼 | 재료 목록 확정 후 Step 2로 이동 |

### 백엔드 API
| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/api/analyze-image` | POST | 이미지를 받아 재료 목록 반환 |

**요청 형식**
```json
{
  "image": "base64 인코딩된 이미지 문자열"
}
```

**응답 형식**
```json
{
  "ingredients": ["계란", "우유", "당근", "양파", "치즈"],
  "raw_description": "모델이 생성한 원문 설명"
}
```

### AI 프롬프트 설계
```
이 냉장고 사진에서 보이는 모든 식재료를 파악해주세요.
결과는 반드시 다음 JSON 형식으로만 응답하세요:
{"ingredients": ["재료1", "재료2", ...]}
식재료가 불분명하거나 보이지 않으면 빈 배열로 응답하세요.
```

---

## 비기능 요구사항
- API 응답 시간: 10초 이내
- 이미지 전송: base64 인코딩 후 JSON으로 전송
- 에러 처리: 이미지 용량 초과, API 오류, 재료 미인식 시 사용자 친화적 메시지 표시

---

## 기술 스택
| 구분 | 기술 |
|------|------|
| 프론트엔드 | HTML, CSS, Vanilla JS (또는 React) |
| 백엔드 | Node.js + Express |
| AI 모델 | `google/gemma-3-27b-it:free` via OpenRouter |
| 환경변수 | `dotenv` (.env의 OPENROUTER_API_KEY) |

---

## 화면 플로우
```
[이미지 업로드 영역]
        ↓ (파일 선택 또는 드래그&드롭)
[이미지 미리보기 + "재료 인식" 버튼]
        ↓ (버튼 클릭 → 로딩)
[인식된 재료 태그 목록 + 편집 UI]
        ↓ (확정)
[→ Step 2: 레시피 생성]
```

## 완료 기준 (Definition of Done)
- [ ] 이미지 업로드 및 미리보기가 동작한다.

- [ ] 재료 목록이 화면에 태그로 표시된다.
- [ ] 재료 추가/삭제 편집이 가능하다.
